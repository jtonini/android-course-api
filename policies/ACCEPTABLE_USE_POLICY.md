# Spiderweb File Upload API - Acceptable Use Policy

**Course:** Android Programming (CSCI XXX)  
**Instructor:** Professor Shweta Ware  
**Semester:** Spring 2026

---

## Purpose

This API provides a learning environment for students to practice HTTP POST and GET operations with file uploads and downloads. The system is designed for educational purposes and must be used responsibly.

---

## Acceptable Use

By using the Spiderweb File Upload API, you agree to:

### 1. Data Usage
- **Use ONLY test/dummy data** - Never upload:
  - Personal information (yours or others')
  - Real student records or grades
  - Sensitive or confidential information
  - Copyrighted material you do not own
  - Any data subject to FERPA, HIPAA, or other privacy regulations

- **Appropriate test data includes:**
  - Sample text files provided by the instructor
  - Dummy JSON or CSV data you create for testing
  - Test images or documents with no personal content
  - Generated or synthetic data for your assignments

### 2. API Token Security
- **Keep your token private** - Do not share it with other students
- **Protect your token** - Do not:
  - Post your token in public forums or repositories
  - Email your token to others
  - Hardcode your token in code you submit for grading
- **Use proper configuration** - Store your token in:
  - BuildConfig fields (recommended)
  - Local properties files (not committed to version control)
  - Secure resource files

### 3. Course-Only Usage
- Use the API **only for course assignments** as directed by your instructor
- Do not use the API for:
  - Personal projects unrelated to the course
  - Testing non-course applications
  - Any commercial purposes
  - Sharing with students not enrolled in this course

### 4. Respectful System Usage
- **Respect storage limits:**
  - Maximum 500 MB per student
  - Maximum 50 MB per file
  - Delete test files when no longer needed

- **Respect rate limits:**
  - Maximum 10 uploads per minute
  - Do not attempt to circumvent limits

- **Do not attempt to:**
  - Access other students' files or directories
  - Probe for security vulnerabilities
  - Upload malicious code or malware
  - Interfere with other students' work
  - Overwhelm the system with excessive requests

### 5. Academic Integrity
- Complete your own work
- Do not share your token to help others bypass authentication
- Do not access files that are not yours
- Follow all University of Richmond Honor Code policies

### 6. Reporting
- **Report security concerns immediately** to:
  - Your instructor (Professor Ware: sware@richmond.edu)
  - Academic Research Computing (João Tonini: jtonini@richmond.edu)
- Report any suspicious activity or potential breaches
- Report any accidental uploads of sensitive data immediately

---

## Consequences of Misuse

Violations of this Acceptable Use Policy may result in:

1. **Immediate token revocation** - Loss of API access
2. **Academic integrity action** - As per University Honor Code
3. **Course grade penalties** - As determined by the instructor
4. **University disciplinary action** - For serious violations
5. **Legal action** - For violations involving illegal activity

---

## System Monitoring

Please be aware that:

- All API activity is **logged** with timestamps
- Logs include: NetID, filenames, file sizes, upload/download times
- Your instructor has access to an **admin dashboard** showing all activity
- System administrators may review logs for security and troubleshooting
- FERPA protections apply to your educational records, including these logs

---

## Data Retention

- Student files are stored for the duration of the semester
- Files will be **deleted 67 days after the semester ends**
- You will receive a **30-day warning** before deletion
- **Download and save any files you wish to keep** before the deletion date
- The University is not responsible for data loss after the retention period
